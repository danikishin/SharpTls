# Code of Conduct

SharpTls is a technical community built around careful, evidence-based collaboration.
Everyone participating in the repository is expected to help keep it useful and safe.

## Expected behavior

- Be respectful, direct and specific about code and protocol behavior.
- Assume good intent while still requiring reproducible evidence.
- Welcome questions and explain specialized TLS or cryptography context.
- Credit prior work and preserve licenses and provenance.
- Protect private information, credentials, captures and vulnerability details.

## Unacceptable behavior

Harassment, discrimination, threats, sexualized attention, deliberate intimidation,
doxxing, sustained disruption, or publishing another person's private information is
not acceptable. Technical disagreement is welcome; personal attacks are not.

## Enforcement

Repository maintainers may edit or remove contributions and may temporarily or
permanently restrict participation when conduct is harmful. Report conduct concerns
privately to the maintainers through a private repository contact channel. Do not use a
public issue when a report contains personal or security-sensitive information.

Maintainers will consider context, protect reporters where possible and respond in a
way proportionate to the behavior and its impact.
